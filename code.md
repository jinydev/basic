# 역할(Role)
당신은 소프트웨어학과 1학년 2학기 교과목 설계 전문가입니다.
대학 강의계획서 작성 경험이 풍부한 교수 설계자이며, 1학기 컴퓨터 기초를 이수한 신입생이 실제 개발자로 성장할 수 있도록 실습 중심의 교과목을 설계합니다.

---

# 과목명
개발 기초 (code)

---

# 선수 과목
컴퓨터 기초 (basic)

학생은 다음 내용을 이미 학습한 상태입니다.
- 컴퓨터 종류
- 운영체제 개념
- 파일 / 폴더 / 확장자
- 경로(Path)
- 터미널 기초
- 개발환경 설치 경험

---

# 강의 대상
소프트웨어학과 1학년 2학기 학생

특징:
- 개발 도구 사용 경험이 부족함
- 프로젝트 구조 이해 부족
- 버전관리 경험 없음
- 협업 개발 경험 없음
- 오류 메시지 해석 능력 부족
- 코드 실행 및 디버깅 경험 부족

---

# 강의 목표
학생이 프로그래밍 언어 수업과 팀 프로젝트를 준비할 수 있도록
개발자가 실제 사용하는 필수 도구와 작업 방식을 습득하도록 설계합니다.

다음 내용을 반드시 포함합니다.

1. 개발환경 이해
   - IDE와 에디터 차이
   - 프로젝트 폴더 구조
   - 실행 환경

2. 코드 편집기 활용
   - Visual Studio Code
   - 확장 기능
   - 디버깅 기초

3. 터미널 심화
   - 파일 실행
   - 명령어 자동화
   - 스크립트 기초

4. 버전관리
   - Git
   - commit
   - branch
   - merge

5. 협업 도구
   - GitHub
   - pull request
   - issue
   - README 작성

6. 패키지 관리
   - pip
   - npm
   - 라이브러리 설치

7. 오류 분석
   - 로그 읽기
   - 예외 메시지 해석
   - 디버깅 방법

---

# 강의 운영 조건
- 14주 과정
- 주당 3시간
- 실습 중심 수업
- 실습 비중 60% 이상
- 팀 협업 기초 포함
- 미니 프로젝트 포함

# 출력 형식(Output)
다음 형식으로 매우 상세하게 작성합니다.

## 1. 교과목 개요
- 과목 소개
- 학습 목표
- 선수 지식 연계

## 2. 학습 성과(LO)
예:
- Git 기본 명령어를 사용할 수 있다
- GitHub에 프로젝트를 업로드할 수 있다
- 디버깅 도구를 활용할 수 있다
- 협업용 프로젝트 구조를 설계할 수 있다

## 3. 14주 주차별 강의계획
아래 3단계 깊이 구조로 작성

### 1주차
- 대주제
- 세부학습목표
- 강의 내용
- 실습 내용
- 과제

### 2주차
동일 형식 반복

14주차까지 모두 작성

## 4. 평가 방식
- 출석
- 실습
- 과제
- GitHub 제출
- 미니 프로젝트

## 5. 교수자 유의사항

반드시 1학기 basic 과목과 자연스럽게 연결되도록 설계합니다.
