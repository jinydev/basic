# 컴퓨터 구조 (computer) 강의계획서

## 1. 교과목 개요

- **과목 소개**: **소프트웨어 전공자**가 좋은 코드를 짜고 시스템을 설계하기 위해 반드시 알아야 할 하드웨어 지식을 다루는 전공 핵심 과목입니다. "CPU 내부 회로를 설계하는 방법"이 아니라 **"CPU·메모리·저장장치가 어떻게 동작하고 그것이 내 코드 성능에 어떤 영향을 주는가"** 를 배웁니다. 오늘날 개발자가 직접 다루는 현대 하드웨어(DDR5·NVMe·PCIe·GPU·AI 가속기)를 실물 중심으로 이해하고, 이후 CPU 원리·메모리·ISA를 **소프트웨어 개발 맥락**으로 연결합니다. 과목을 마치면 목적에 맞는 컴퓨터를 설계·구매하고 서버·클라우드 인프라를 기초 수준에서 설계할 수 있습니다.
- **학습 순서 원칙**: ① 현대 실물 하드웨어 → ② 목적별 설계·구매 → ③ 데이터 표현 이론 → ④ CPU·메모리 원리(SW 관점) → ⑤ AI 가속기 시스템 → ⑥ 인프라·서버 설계 → ⑦ 레거시 역사 + 미래 기술
- **이 과목의 깊이**: 하드웨어를 이해하는 소프트웨어 개발자 수준을 목표로 합니다. 회로 설계(Verilog·FPGA)나 논리 게이트 합성은 이 과목의 범위가 아닙니다. 캐시 최적화 프로그래밍·어셈블리 분석·SIMD·GPU 커널 등 **하드웨어를 코드로 제어하는 심화 역량**은 후속 과목 <컴퓨터 구조 심화(computer2)>에서 다룹니다.

---

## 2. 학습 성과(LO)

- 최신 PC 스펙을 보고 부품별 역할과 수치의 의미를 해석할 수 있다.
- 게이밍·개발·AI 워크스테이션·서버 등 목적별 컴퓨터를 스스로 설계하고 구매 선택을 논리적으로 정당화할 수 있다.
- PC 조립의 전체 단계를 이해하고 부품 호환성을 확인할 수 있다.
- 멀티코어·GPU 병렬 연산·AI 가속기의 설계 외도와 작동 원리를 설명할 수 있다.
- 서버·NAS·클라우드 인프라의 기초 구성 요소를 도식으로 설명할 수 있다.
- 레거시 기술이 현재 기술로 대체된 이유를 설명하고, 미래 컴퓨팅 방향을 개념 수준에서 논의할 수 있다.

### 1주차
- **대주제**: 현대 PC 해부와 조립 체험 — 부품을 눈으로 보고 손으로 만지다
- **세부학습목표**: 오늘날 PC의 실제 부품 구성을 실물·영상으로 파악하고, PC 조립의 전체 흐름을 순서대로 이해한다.

#### 📌 1-1. 현대 PC 부품 구성과 전원(PSU)의 이해
1. CPU — 메인보드 소켓 위에 장착하는 두뇌, Intel LGA1851·AMD AM5 소켓 구분
2. RAM — DDR5 DIMM 슬롯에 꽂는 임시 기억 공간, 이중 채널 슬롯 순서 주의
3. M.2 NVMe SSD — 메인보드에 직접 꽂는 초소형 저장장치, SATA·NVMe 규격 구분
4. GPU — PCIe x16 슬롯에 장착하는 그래픽 카드, 보조 전원(8핀·16핀) 연결 필요
5. 파워 서플라이(PSU) 전원 공급 — 일반 PC 전원의 종류(ATX, SFX)와 구성, 모듈러 케이블, 80PLUS 효율 등급·용량(W) 선택 기준

#### 📌 1-2. PC 조립 순서와 시스템 초기화(BIOS/UEFI)
1. PCPartPicker.com — 부품 호환성 자동 검사 사이트 (소켓·TDP·케이스 크기 등)
2. 조립 순서 ① CPU → ②쿨러 → ③ RAM → ④ SSD → ⑤ 메인보드 케이스 고정 → ⑥ GPU → ⑦ PSU 연결
3. 처음 부팅 시 POST(Power-On Self Test) — 하드웨어 이상 유무를 점검하는 부팅 성공·실패 신호음 의미
4. BIOS/UEFI의 역할과 설정 — 기본 입출력 시스템(BIOS)의 개념, 메인보드 펌웨어 진입 및 조작법
5. BIOS 필수 설정 항목 — RAM 오버클럭(XMP/EXPO), 부팅 드라이브 순서(Boot Order), 쿨링팬 속도 곡선, 가상화(VT-x/SVM) 활성화

#### 📌 1-3. 내 PC 사양 직접 읽기
1. CPU-Z(Windows) — CPU·메모리·메인보드 실시간 스펙 조회 무료 도구
2. `system_profiler SPHardwareDataType`(macOS) — 터미널로 하드웨어 정보 전체 출력
3. `lshw -short`(Linux) — 장치 계층 구조 전체 목록 출력
4. 제조사 스펙시트에서 읽어야 할 지표 — 코어 수·클록·TDP·L3 캐시·PCIe 레인 수
5. HWiNFO64 — 온도·전압·팬 속도를 실시간으로 모니터링하는 고급 진단 도구

- **강의 내용**: 현대 PC 부품 실물 영상, 조립 순서·호환성, BIOS POST, 진단 도구
- **실습 내용**: PCPartPicker로 예산 100만원 PC 부품 목록 조립, CPU-Z / HWiNFO64로 사양 전체 캡처
- **과제**: 본인 PC 사양표(CPU·RAM·SSD·GPU) + PCPartPicker로 동급 타 조합 비교 리포트 제출

### 2주차
- **대주제**: 현대 CPU — 멀티코어 아키텍처와 성능의 비밀
- **세부학습목표**: 싱글코어 한계 이후 멀티코어·이종코어(하이브리드) 시대로 전환된 이유를 이해하고, Intel·AMD·Apple의 최신 아키텍처를 비교한다.

#### 📌 2-1. 싱글코어 시대의 종말과 멀티코어의 등장
1. 클록 속도 한계 — 2004년경 GHz 경쟁이 멈춘 이유: 발열·전력 소비의 물리 한계
2. 덴나드 스케일링 붕괴 — 공정 미세화가 더 이상 자동 속도 향상을 보장하지 못하는 이유
3. 멀티코어 전환 — "더 빠른 하나" 대신 "여러 개를 동시에" 전략으로 전환
4. 코어(Core)와 스레드(Thread) — 물리 코어와 하이퍼스레딩(SMT)으로 논리 코어 2배 효과
5. 멀티코어를 활용하려면 소프트웨어도 병렬로 써야 하는 이유 — 암달의 법칙(Amdahl's Law)

#### 📌 2-2. 현대 CPU 이종코어(Hybrid) 설계
1. Intel Core Ultra P·E·LP-E 코어 구분 — 성능(P)·효율(E)·저전력 효율(LP-E) 역할 분담
2. AMD Ryzen의 CCD·IOD Chiplet 설계 — 여러 소형 다이를 이어붙여 단일 CPU 구현
3. Apple M 시리즈 SoC — CPU·GPU·NPU·메모리를 하나의 패키지에 통합
4. ARM big.LITTLE(high-performance + efficiency 코어) — 스마트폰에서 이종코어의 시작
5. 스케줄러(OS)가 어떤 작업을 P코어 vs E코어에 배분하는지 결정하는 원리

#### 📌 2-3. CPU 핵심 성능 지표 해석
1. 클록 속도(Base/Boost GHz) — 기본 클록과 터보 부스트 클록, 온도에 따른 쓰로틀링
2. L1·L2·L3 캐시 — 레벨별 크기·속도 트레이드오프, 게임·AI 작업에서의 영향 차이
3. TDP(열설계전력) — 냉각 시스템(공랭·수랭·TEC) 선택의 기준
4. 공정 노드(nm) — 3nm·4nm가 성능·전력 효율에 가져오는 실질적 이점
5. Cinebench R23·Geekbench 6 — 싱글·멀티코어 성능 측정으로 실제 서열 확인

- **강의 내용**: 멀티코어 전환 이유, 이종코어 설계 비교, CPU 스펙 해석
- **실습 내용**: Cinebench R23으로 내 CPU 싱글·멀티코어 점수 측정, 동급 CPU 순위 비교
- **과제**: AMD·Intel·Apple CPU 3종 각각의 특성을 비교한 표 작성 + 용도별(게임·개발·AI) 추천 근거 제출

### 3주차
- **대주제**: 현대 메모리 시스템 — DDR5·LPDDR5·HBM과 메모리 계층
- **세부학습목표**: 현재 PC·노트북·AI 가속기에서 사용 중인 메모리 기술을 이해하고, 메모리 계층이 소프트웨어 성능에 미치는 영향을 체험한다.

#### 📌 3-1. 기억장치의 종류와 동작 원리 (ROM, RAM, DRAM)
1. ROM(Read Only Memory)과 비휘발성 — 과거 BIOS 저장부터 현대의 EEPROM/플래시 메모리까지의 역할
2. RAM(Random Access Memory)의 의미 — 순차 접근(테이프)과 대비되는 임의 접근 메모리의 개념
3. SRAM vs DRAM — 캐시에 쓰이는 SRAM(플립플롭 구조)과 메인 메모리에 쓰이는 DRAM(축전기 리프레시 구조)의 동작원리 차이
4. 메모리의 발전 역사 — FPM/EDO DRAM → SDRAM → DDR 시대로의 진화 과정
5. 현대 DDR5 규격 — DDR4 대비 2배 대역폭, 4800~8000MHz, 온 다이 ECC 탑재 및 이중 채널 구성

#### 📌 3-2. HBM과 AI 메모리
1. HBM(High Bandwidth Memory) — DRAM 다이를 수직 적층하여 초고 대역폭 실현
2. HBM3e 스펙 — 1.2TB/s+ 대역폭이 AI 행렬 연산 병목 해결에 결정적인 이유
3. GDDR7(게이밍 GPU) vs HBM(AI GPU) — 용량·속도·비용 트레이드오프
4. Apple Unified Memory — CPU와 GPU가 동일 메모리 풀을 공유하여 복사 비용 제거
5. 인메모리 컴퓨팅(Processing-In-Memory) — 메모리 내부에서 직접 연산하는 미래 방향

#### 📌 3-3. 메모리 계층 구조
1. 레지스터(<1ns) → L1(1ns) → L2(4ns) → L3(10ns) → DDR5(60ns) → NVMe(수십µs) 접근 시간 비교표
2. 캐시 지역성 — 시간적(최근 데이터 재접근)·공간적(인접 주소 연속 접근) 두 가지 패턴
3. 캐시 미스(Cache Miss)가 소프트웨어 성능에 미치는 영향 — 순차 vs 랜덤 접근 실험
4. NUMA(Non-Uniform Memory Access) — 멀티소켓 서버에서 메모리 거리 차이 문제
5. 메모리 용량별 작업 가능 범위 — 8GB·16GB·32GB·64GB·192GB 각각 무엇이 가능한가

- **강의 내용**: DDR5·LPDDR5·HBM 규격, 메모리 계층 접근 시간, 캐시 지역성
- **실습 내용**: Python numpy로 순차 접근 vs 무작위 접근(1억 원소) 실행 시간 비교 실험
- **과제**: 실험 결과표 + 내 PC Ram 용량으로 처리할 수 있는 작업 한계 분석 리포트

### 4주차
- **대주제**: 현대 저장장치 — NVMe·USB·클라우드
- **세부학습목표**: 오늘날 실제로 사용되는 저장장치의 구조와 성능을 이해하고, 용도별 적합한 스토리지를 선택할 수 있다.

#### 📌 4-1. SSD 저장매체 종류와 컨트롤러 구조
1. SSD의 탄생과 인터페이스 변천사 — 물리 헤드가 없는 플래시 스토리지의 등장, SATA부터 PCIe/NVMe까지의 대역폭 진화
2. NAND Flash 셀 구조 — SLC(1비트)·MLC(2비트)·TLC(3비트)·QLC(4비트)의 용량·속도·수명(TBW) 트레이드오프 비교
3. 모바일 저장장치 요소 — 스마트폰/가벼운 기기용 eMMC와 UFS의 속도 및 차이점
4. SSD 컨트롤러 시스템 — 웨어 레벨링(마모 평준화), 가비지 컬렉션, DRAM 캐시를 통해 수명과 속도를 관리하는 두뇌
5. 고속 스토리지 체감 — PCIe 4.0/5.0 기준 7,000MB/s+ 읽기 속도가 프로그램 로딩 및 대용량 처리에 미치는 영향

#### 📌 4-2. 외부 인터페이스와 휴대용 저장
1. USB 버전별 속도 — USB 3.2 Gen2(10Gbps)·USB4(40Gbps)·USB4 v2.0(80Gbps) 실사용 차이
2. Thunderbolt 4·5 — 인증된 초고속 외장 SSD(10GB/s+), 모니터·충전·데이터 통합
3. USB-C 커넥터 혼란 해결 — 같은 형태인데 지원 규격이 다른 이유와 확인 방법
4. SD카드 규격 — UHS-I·UHS-II·SD Express, 카메라·드론 작업 시 속도 한계
5. 외장 NVMe SSD 케이스 — M.2 SSD를 USB4/Thunderbolt 케이스에 넣어 휴대 고성능 스토리지

#### 📌 4-3. 클라우드 저장과 분산 스토리지
1. 개인 NAS(Synology·QNAP) — 홈 클라우드 구성, RAID 개념과 데이터 보호
2. 오브젝트 스토리지(Amazon S3·Google GCS) — 파일 시스템과 다른 무한 확장 키-값 스토리지
3. AI 학습용 대용량 스토리지 — 수십 PB 데이터셋 처리를 위한 분산 파일 시스템
4. NVMe over Fabrics(NVMe-oF) — 네트워크를 통해 원격 NVMe를 로컬처럼 사용하는 기업 기술
5. 콜드·웜·핫 스토리지 계층 — 접근 빈도에 따라 비용이 달라지는 클라우드 스토리지 전략

- **강의 내용**: NVMe 구조, USB·Thunderbolt 버전별 비교, 클라우드·NAS 스토리지
- **실습 내용**: 내 SSD 실측 속도 측정 + NAS·S3 저장 구조 비교 도식 작성
- **과제**: 내 SSD 스펙(TBW) 기반 남은 수명 추산 리포트 + 용도별 저장장치 선택 가이드 작성

### 5주차
- **대주제**: 현대 인터페이스와 버스 — PCIe·Thunderbolt·Wi-Fi 7
- **세부학습목표**: 오늘날 사용되는 내외부 인터페이스의 대역폭을 이해하고, 장치 연결 시 병목 지점을 파악하는 능력을 기른다.

#### 📌 5-1. PCIe — 내부 고속 버스
1. PCIe 버전별 대역폭 — 4.0(16GT/s)·5.0(32GT/s)·6.0(64GT/s) per lane 비교
2. x1·x4·x8·x16 슬롯 — 레인 수에 따른 총 대역폭 계산법
3. PCIe Bifurcation — 1개의 x16 슬롯을 x8+x8 또는 x4×4로 나누는 방법
4. GPU와 NVMe가 PCIe 레인을 어떻게 나눠 쓰는가 — CPU·칩셋 레인 배분 구조
5. CXL(Compute Express Link) — CPU·가속기·메모리 확장을 캐시 일관성 있게 연결하는 차세대 표준

#### 📌 5-2. 입력 장치 인터페이스의 역사와 종류
1. 키보드의 진화 — 펀치카드부터 기계식, 멤브레인, 펜타그래프, 정전용량 무접점까지의 특징과 원리
2. 마우스의 탄생과 역사 — 더글라스 엥겔바트의 목재 마우스, 볼 마우스에서 광학/레이저 마우스로의 발전
3. 트랙패드와 제스처 — 노트북 트랙패드의 등장, 멀티터치 제스처 처리 로직
4. 햅틱(Haptic) 인터페이스 — 진동 모터와 리니어 모터(Taptic Engine)를 이용해 물리적 클릭감을 가상으로 구현하는 원리
5. 현대 외부 I/O 인터페이스 연장 — USB4, Thunderbolt, Bluetooth 5.x를 통한 초고속 및 무선 입력기기 연결성

#### 📌 5-3. 출력 장치: 모니터 변천사와 해상도
1. 모니터 기술 역사 — 두꺼운 CRT(브라운관) → 얇은 LCD/LED → 차세대 OLED 및 MicroLED의 색 재현율과 명암비(HDR) 발전
2. 디스플레이 해상도 규격 — HD(720) → FHD(1080) → QHD(1440) → 4K(2160) → 8K 픽셀 밀도(ppi)와 선명도 비교
3. 주사율(Refresh Rate)과 응답속도 — 60Hz에서 144Hz/240Hz로 게이밍 스크린 경험이 어떻게 달라지는가
4. 영상 출력 포트 역사 — VGA(아날로그) → DVI → HDMI → DisplayPort로의 디지털 인터페이스 전송 대역폭 확장
5. 서버·네트워크 출력 개요 — 콘솔 출력 및 고대역폭 이더넷(10GbE), NVLink 소개 (네트워크 장치로서의 I/O)

- **강의 내용**: PCIe 세대별 비교, 외부 인터페이스(USB·Thunderbolt·HDMI·Wi-Fi 7), 서버 네트워크
- **실습 내용**: 본인 PC PCIe 슬롯 구성과 USB 버전을 CPU-Z/`lspci`로 확인, USB 전송 속도 실측
- **과제**: 자신의 PC 인터페이스 구성표 작성 + 병목 지점 분석 리포트

### 7주차
- **대주제**: 데이터의 표현 — 비트·바이트·진수·AI 데이터 타입
- **세부학습목표**: 현대 하드웨어 이해의 기반이 되는 데이터 표현 체계와 AI 시대의 저정밀도 숫자 포맷을 스펙 수치와 연결하여 이해한다.

#### 📌 7-1. 비트·진수 체계
1. 비트(bit)와 바이트(Byte) — 1B=8b, KB·MB·GB·TB·PB 단위 체계와 기가 계산
2. 2진수 — 전압의 높낮이(0·1)로 모든 데이터를 표현하는 물리적 기반
3. 16진수(Hex) — 메모리 주소·색상 코드·MAC 주소에서 2진수를 간결하게 표현
4. 2·10·16진수 상호 변환 규칙과 Python `bin()`, `hex()`, `int()` 활용
5. 단위 접두사 혼동 — SI 단위(kilo=1000) vs 이진 단위(kibi=1024)의 차이, 저장장치 용량 표기 차이

#### 📌 7-2. 정수·실수 표현과 오버플로우
1. 부호 없는 정수(unsigned) vs 부호 있는 정수(signed) — 2의 보수(Two's Complement) 원리
2. 데이터 타입 크기 — int8(−128~127)·int32·uint64 범위 와 오버플로우 체험
3. IEEE 754 부동소수점 — 부호·지수·가수 3구역 구조, `0.1+0.2≠0.3` 원인
4. FP32(float32) — 전통 딥러닝 학습 표준 / FP16(half precision) — 속도 2배 향상
5. BF16(Brain Float 16) — Google이 AI 연산을 위해 설계한 지수 범위 우선 포맷

#### 📌 7-3. 문자 인코딩과 양자화
1. ASCII — 영문·숫자 128자를 7비트로 매핑한 최초 표준, 컴퓨터가 영어 친화적인 뿌리
2. UTF-8 — 가변 길이로 전 세계 문자 표현, 한글 3바이트·이모지 4바이트
3. 토크나이저 — LLM이 텍스트를 숫자 ID 시퀀스로 변환하는 방식(BPE 개요)
4. INT8·INT4 양자화(Quantization) — 모델 파라미터 정밀도를 낮춰 메모리·속도 최적화
5. 양자화의 정확도 손실 — FP32 → INT4 변환 시 발생하는 품질 저하 실제 사례

- **강의 내용**: 비트·진수 체계, 부동소수점, AI 데이터 타입(FP16·BF16·INT8·INT4)
- **실습 내용**: Python으로 float32 vs float16 정밀도 손실 비교, GPT-2 117M 모델 저장 용량 계산
- **과제**: AI 모델 파라미터를 FP32·FP16·INT8로 저장할 때 필요한 메모리 용량 직접 계산 제출

### 9주차
- **대주제**: CPU 내부 원리 — 논리 게이트·ALU·명령어 사이클
- **세부학습목표**: 현대 CPU가 실제 연산을 수행하는 내부 메커니즘을 시뮬레이터로 추적하며, 소프트웨어가 결국 전기 신호로 실행되는 원리를 이해한다.

#### 📌 9-1. 디지털 논리 게이트 기초
1. 트랜지스터의 스위칭 동작 — 전압의 높낮이가 논리 0·1로 추상화되는 물리적 기반
2. AND·OR·NOT·XOR·NAND·NOR — 진리표와 회로 기호, NAND의 범용성
3. 반가산기(XOR+AND) — 1비트 덧셈과 올림수를 동시에 처리하는 최소 연산 회로
4. 전가산기(Full Adder) — 이전 자리 올림수까지 포함한 완전한 1비트 덧셈 회로
5. CircuitVerse.org — 브라우저에서 논리 게이트를 직접 배선하는 무료 시뮬레이터

#### 📌 9-2. ALU와 레지스터
1. ALU(Arithmetic Logic Unit) — CPU 내부의 사칙연산·비교·논리 연산 전담 유닛
2. 레지스터 — CPU 내부 초고속 임시 저장 공간 (일반·인덱스·플래그·프로그램 카운터)
3. CU(Control Unit) — 명령어를 해석하고 ALU·레지스터·메모리에 제어 신호 분배
4. 레지스터가 RAM보다 수백 배 빠른 이유 — 물리적 거리와 전용 회로의 차이
5. Ripes.me — 브라우저에서 어셈블리 명령어를 스텝 단위로 실행하며 레지스터 추적

#### 📌 9-3. 명령어 사이클과 파이프라이닝
1. Fetch — PC가 가리키는 메모리 주소에서 명령어 인출
2. Decode — CU가 명령어를 해석하여 어떤 연산·피연산자인지 파악
3. Execute — ALU가 연산하고 결과를 레지스터·메모리에 저장
4. 파이프라이닝 — Fetch·Decode·Execute를 겹쳐 실행하여 처리량(Throughput) 향상
5. 분기 예측(Branch Prediction)·비순서 실행(Out-of-Order) — 현대 CPU 성능 극대화 기법

- **강의 내용**: 논리 게이트·ALU·레지스터·CU 구조, Fetch-Decode-Execute 사이클, 파이프라이닝
- **실습 내용**: CircuitVerse에서 반가산기 구성, Ripes에서 ADD·MOV·CMP 스텝 추적
- **과제**: Ripes에서 명령어 5줄 실행 후 각 단계별 레지스터 상태 표 정리 제출

### 10주차
- **대주제**: ISA·GPU 병렬 연산 — RISC·ARM·SIMD·Tensor Core
- **세부학습목표**: 명령어 세트 설계 철학과 GPU 병렬 연산 원리를 이해하고, 현재 ARM의 독주와 RISC-V의 부상을 시장 맥락으로 분석한다.

#### 📌 10-1. ISA 설계 철학: RISC vs CISC
1. ISA(Instruction Set Architecture) — 소프트웨어가 CPU에게 명령하는 공식 인터페이스
2. CISC(x86) — 복잡하고 가변 길이 명령어, 코드 밀도 우선, PC 역사의 유산
3. RISC(ARM) — 단순·균일 명령어, 파이프라이닝·전력 효율 최적화
4. 현대 x86 내부는 RISC — CISC 명령어를 내부에서 µop로 분해하여 실행하는 구조
5. RISC-V — 오픈소스 ISA, 로열티 제로, 중국 반도체·AI 칩 독립 전략에서의 역할

#### 📌 10-2. GPU 병렬 연산의 물리 원리
1. CPU vs GPU 설계 철학 — 소수 강력 코어(낮은 지연) vs 수천 소형 코어(높은 처리량)
2. SIMD(Single Instruction, Multiple Data) — 하나의 명령어로 수천 데이터를 동시 처리
3. CUDA 코어 vs Tensor Core — 범용 연산 vs AI 행렬 곱 전용 가속 유닛
4. Warp — 32개 스레드가 동시에 같은 명령을 실행하는 GPU 스케줄 단위
5. VRAM 대역폭의 중요성 — 연산 유닛이 빠르더라도 메모리 I/O가 병목이 되는 이유

#### 📌 10-3. 현대 GPU 제품과 실험
1. NVIDIA Ada Lovelace(RTX 4000) — CUDA·Tensor·RT Core 3종 구조
2. AMD RDNA 4(RX 9000) — FSR 4 AI 업스케일링, Compute Unit 구조
3. Apple Silicon GPU — Unified Memory·Metal 2 API, macOS 전용 최적화
4. Google Colab T4 GPU — 무료로 GPU 가속을 경험하는 클라우드 환경
5. CPU numpy vs GPU cupy — 행렬 곱(10000×10000) 속도 비교 실험

- **강의 내용**: RISC vs CISC 철학, ARM·RISC-V 현황, GPU 병렬 원리, 현대 GPU 비교
- **실습 내용**: Google Colab에서 CPU·GPU 행렬 곱 속도 비교 실험
- **과제**: Apple M4 vs x86 전력 효율 차이를 ISA 설계 관점으로 분석한 리포트

## 5. computer2 과목 연계 안내

> 이 과목(computer)은 개발자에게 필수적인 **하드웨어 전반의 폭넓은 이해**를 목표로 설계되었습니다.
> 다음 심화 과목 <컴퓨터 구조 심화(computer2)>에서는 하드웨어를 코드로 깊이 있게 제어하고 성능을 최적화하기 위해 아래 주제를 다룹니다:
>
> - **CPU 수준의 실행 원리** (슈퍼스칼라·비순서/투기적 실행, 인라인 어셈블리 읽기)
> - **컴파일러 구조와 코드 최적화** (IR 패스, 루프 언롤링, PGO/LTO)
> - **메모리 계층 프로그래밍** (캐시 친화성, False Sharing, C++11 메모리 모델과 원자 연산)
> - **SIMD 벡터 연산과 병렬 프로그래밍** (AVX/NEON 인트린식, OpenMP)
> - **시스템 수준 I/O 제어** (시스템 콜 최적화, mmap, io_uring, epoll)
> - **GPU 컴퓨팅 성능 제어** (CUDA 커널, 워프/메모리 합병 최적화, Roofline 벤치마킹)
> - **임베디드와 언어 런타임** (FreeRTOS 멀티태스크 최적화, Python/JVM의 JIT 및 GC 하드웨어 오버헤드)
