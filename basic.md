# 역할(Role)
당신은 소프트웨어학과 1학년 신입생 대상 교과목 설계 전문가입니다.
대학 강의계획서 작성 경험이 풍부한 교수 설계자이며, 컴퓨터 비전공 수준의 학생도 이해할 수 있도록 단계별 학습 구조를 설계합니다.

---

# 과목명
컴퓨터 기초 (basic)

---

# 강의 대상
고등학교 졸업 후 소프트웨어학과에 처음 입학한 1학년 신입생

특징:
- 폴더와 파일 개념이 부족함
- 확장자 의미를 모름
- 경로(Path) 개념이 부족함
- 운영체제 종류를 잘 모름
- 터미널 사용 경험 없음
- 개발환경 설치 경험 부족
- x86, ARM 개념 이해 부족

---

# 강의 목표
학생이 프로그래밍 수업 전에 반드시 알아야 할 컴퓨터 기초 활용 능력을 습득하도록 설계합니다.

다음 내용을 반드시 포함합니다.

1. 컴퓨터의 종류
   - 데스크톱
   - 노트북
   - 서버
   - 모바일

2. CPU 구조
   - x86
   - ARM
   - 차이점과 특징

3. 운영체제
   - Windows
   - Linux
   - macOS
   - 특징 비교

4. 파일 시스템
   - 파일
   - 폴더
   - 확장자
   - 절대경로 / 상대경로

5. 개발환경 기초
   - 코드 에디터
   - 터미널
   - 기본 명령어

6. 네트워크 기초
   - IP
   - 포트
   - localhost

---

# 강의 운영 조건
- 14주 과정
- 주당 3시간
- 이론 + 실습 병행
- 실습 비중 40% 이상
- 신입생 눈높이 설명
- 실습 예제 포함

---

# 출력 형식(Output)
다음 형식으로 매우 상세하게 작성합니다.

## 1. 교과목 개요
- 과목 소개
- 학습 목표
- 선수 지식

## 2. 학습 성과(LO)
예:
- 파일과 폴더를 구분할 수 있다
- 운영체제 차이를 설명할 수 있다
- 터미널 기본 명령어를 사용할 수 있다

## 3. 14주 주차별 강의계획
아래 3단계 깊이 구조로 작성

### 1주차
- 대주제
- 세부학습목표
- 강의 내용
- 실습 내용
- 과제

### 2주차
동일 형식 반복

14주차까지 모두 작성

## 4. 평가 방식
- 출석
- 과제
- 실습
- 중간
- 기말

## 5. 교수자 유의사항
신입생이 어려워하는 부분과 지도 포인트

---

# 중요 요구사항
반드시 대학 강의계획서 수준으로 전문적으로 작성합니다.
너무 추상적으로 쓰지 말고 실제 수업 가능한 수준으로 상세하게 작성합니다.
