# 컴퓨터 기초 (basic) 강의계획서

## 1. 교과목 개요

- **과목 소개**: 컴퓨터를 거의 사용해본 경험이 없거나 유튜브·SNS 수준에 머물렀던 학생을 위한 **전공 입문 선수 과목**입니다. 이 과목에서는 개발이나 코딩을 배우는 것이 아닙니다. "컴퓨터가 낯설고 두렵다"는 감정을 없애고, 컴퓨터를 내 의지대로 **능동적으로 다룰 수 있는 일반 사용자**가 되는 것이 목표입니다.
- **학습 목표**: 파일과 폴더를 자유롭게 관리하고, OS 환경을 스스로 제어하며, 사용자 계정 구조를 이해하고, 네트워크 기초 개념을 설명할 수 있습니다. "컴퓨터가 이제 좀 친한 것 같아요"가 이 과목의 합격 기준입니다.
- **선수 지식**: 없음. 스마트폰을 사용할 수 있는 수준이면 충분합니다.

## 2. 학습 성과(LO)

- 내 PC의 사양(CPU·RAM·저장장치·OS 버전)을 스스로 확인하고 의미를 해석할 수 있다.
- 파일·폴더·확장자의 의미를 이해하고, 원하는 위치에 직접 이동·복사·삭제할 수 있다.
- 절대경로와 상대경로를 구분하여 말로 설명할 수 있다.
- 사용자 계정의 구조를 이해하고, 한글 계정명이 시스템 오류를 유발하는 이유를 설명할 수 있다.
- 컴퓨터가 영어 친화적으로 설계된 역사적·기술적 이유를 설명할 수 있다.
- IP·포트·localhost의 개념을 일상적인 비유로 설명할 수 있다.

### 1주차
- **대주제**: 우리가 사용하는 컴퓨터는 무엇인가?
- **세부학습목표**: 컴퓨터의 종류별 특징과 내부 부품의 역할을 비유로 이해하고, 내 PC의 사양을 스스로 확인한다.

#### 📌 1-1. 컴퓨터의 종류와 용도
1. 데스크톱 — 성능·확장성 우선, 자리를 고정하여 사용하는 형태
2. 노트북 — 이동성과 성능의 절충, 배터리로 동작하는 설계
3. 태블릿·스마트폰 — 터치 중심 OS, 앱 실행에 최적화된 경량 컴퓨터
4. 서버 — 24시간 연속 운영을 위한 안정성 우선 컴퓨터, 집 밖 인터넷 어딘가에 존재
5. 임베디드(IoT) — 냉장고·자동차 안에 숨어 있는 특정 기능 전용 소형 컴퓨터

#### 📌 1-2. 컴퓨터 핵심 부품 비유로 이해하기
1. CPU — '요리사': 모든 계산과 판단을 처리하는 두뇌
2. RAM — '작업대': 지금 하는 작업 내용이 올라오는 임시 공간 (전원 끄면 사라짐)
3. SSD/HDD — '냉장고': 꺼도 사라지지 않는 영구 저장 공간
4. GPU — '조리 보조원 수백 명': 게임·영상·AI 등 반복 계산을 동시에 처리
5. 메인보드 — '주방 구조': 모든 부품이 연결되는 기반 회로판

#### 📌 1-3. 내 PC 사양 직접 확인하기
1. Windows: 설정 → 시스템 → 정보 — CPU 이름·RAM 용량·OS 버전 확인
2. macOS: Apple 메뉴 → 이 Mac에 관하여 — 칩·메모리·저장 공간 확인
3. CPU 이름 해석법 — Intel Core i5-14세대, Apple M3, AMD Ryzen 7 읽는 법
4. 화면 캡처 방법 — Windows `Win+Shift+S`, macOS `Cmd+Shift+4`
5. 사양이 낮을 때 생기는 현상 — 느린 부팅·버벅임·용량 부족 등

- **강의 내용**: 컴퓨터 종류·부품 역할 비유 설명, 사양 확인 방법
- **실습 내용**: 내 PC 시스템 정보 화면 열기 → 항목별 의미 해석 → 화면 캡처
- **과제**: 내 PC 사양(CPU·RAM·OS 버전·저장 공간) 캡처 후 각 항목 한 줄 해석 제출

### 2주차
- **대주제**: 내 컴퓨터 완전 정복 — 프로그램 설치·제거·단축키
- **세부학습목표**: 프로그램 설치의 구조를 이해하고, 단축키로 빠르게 작업하며, 컴퓨터가 멈출 때 당황하지 않고 대처하는 법을 익힌다.

#### 📌 2-1. 프로그램 설치와 제거의 구조
1. 설치 파일 형식 — `.exe`·`.msi`(Windows), `.dmg`·`.pkg`(macOS) 차이
2. 설치 마법사 진행 시 주의 사항 — 번들 프로그램 체크 해제, 설치 경로 확인
3. 앱 스토어 설치 vs 직접 다운로드 설치의 차이 — 검증 여부와 업데이트 방식
4. 프로그램 제거의 올바른 방법 — 제어판/설정 앱 vs 폴더 직접 삭제 차이
5. 잔여 파일(캐시·레지스트리) 정리가 필요한 이유

#### 📌 2-2. 생산성 필수 단축키
1. 텍스트 작업 — `Ctrl+C/X/V`, `Ctrl+Z/Y`, `Ctrl+A`, `Ctrl+F` (Mac은 Ctrl→Cmd)
2. 창 관리 — `Alt+Tab`(Win) / `Cmd+Tab`(Mac), `Win+D` 바탕화면 보기
3. 스크린샷 — `Win+Shift+S`, `Print Screen`, `Cmd+Shift+3/4`
4. 파일 탐색기 — `F2` 이름 바꾸기, `Delete` / `Shift+Delete` 영구 삭제 차이
5. 브라우저 — `Ctrl+T` 새 탭, `Ctrl+L` 주소창 이동, `Ctrl+R` 새로고침, `Ctrl+W` 탭 닫기

#### 📌 2-3. 시스템 부하와 강제 종료 대처
1. 프로세스(Process) — 실행 중인 프로그램의 독립적인 인스턴스
2. Windows 작업 관리자(`Ctrl+Shift+Esc`) — CPU·메모리 점유율 확인 및 프로세스 강제 종료
3. macOS 활성 상태 보기 — 앱이 응답하지 않을 때 강제 종료하는 방법
4. 시스템이 느려지는 3가지 원인 — RAM 부족·CPU 과부하·저장 공간 꽉 참
5. 재부팅 전 반드시 확인할 것 — 작업 저장 여부, 안전한 종료 순서

- **강의 내용**: 설치 구조, 필수 단축키, 작업 관리자 활용법
- **실습 내용**: 무거운 프로그램 실행 → 작업 관리자에서 점유율 확인 → 강제 종료 체험
- **과제**: 본인 PC에서 불필요한 프로그램 1개 제거하는 과정 캡처 + 단축키 치트시트(15개 이상) 작성 제출

### 3주차
- **대주제**: 사용자 계정 — 내 컴퓨터 안의 또 다른 나
- **세부학습목표**: 컴퓨터의 사용자 계정 구조를 이해하고, 한글 계정명이 오류를 일으키는 이유와 영문 계정 사용의 중요성을 파악한다.

#### 📌 3-1. 사용자 계정의 구조
1. 관리자(Administrator) 계정 — 모든 설치·설정 변경이 가능한 최고 권한 계정
2. 표준(Standard) 계정 — 일상 사용에 적합, 시스템 변경에는 관리자 비밀번호 필요
3. 게스트 계정 — 임시 사용자를 위해 파일 접근이 제한된 제한적 계정
4. Microsoft 계정 vs 로컬 계정 — 온라인 연동 vs 오프라인 독립 사용의 차이
5. 다중 사용자 환경 — 가족 PC에서 각자의 파일과 설정이 분리되는 원리

#### 📌 3-2. 한글 계정명이 오류를 일으키는 이유
1. Windows 사용자 폴더 경로 — `C:\Users\홍길동\Documents` 와 같이 한글이 경로에 포함됨
2. 한글 경로가 문제를 일으키는 상황 — 개발 도구 설치·Python 패키지·게임 실행 오류 사례
3. 영문 경로를 요구하는 소프트웨어들 — npm·Python·Unity·게임 런처 등
4. 계정명 변경의 어려움 — Windows에서 계정명 변경이 폴더 경로를 바꾸지 않는 이유
5. 처음부터 영문 계정으로 만들어야 하는 이유 — 사후 수정이 매우 복잡함

#### 📌 3-3. 사용자 계정 관련 실무 설정
1. 새 영문 계정 생성 방법 — `설정 → 계정 → 가족 및 다른 사용자` 경로
2. 기존 한글 계정에서 새 영문 계정으로 파일 이전 방법
3. UAC(User Account Control) — "이 앱이 변경하도록 허용하시겠습니까?" 팝업의 의미
4. 폴더 권한(읽기·쓰기·실행) — 내 파일인데도 '액세스 거부'가 뜨는 이유
5. 비밀번호 없는 계정의 보안 위험 — 공용 PC에서의 계정 격리 필요성

- **강의 내용**: 계정 종류와 권한 체계, 한글 계정명 오류 원인, 영문 계정의 중요성
- **실습 내용**: 본인 계정명 확인 → `C:\Users\` 경로에 한글이 있는지 점검, 새 영문 계정 생성 실습
- **과제**: 현재 계정명이 한글인 경우 영문 계정으로 전환 계획서 작성, 영문인 경우 계정 권한 구조 설명 제출

### 4주차
- **대주제**: 컴퓨터는 왜 영어를 좋아하는가 — 영어와 컴퓨터의 뿌리
- **세부학습목표**: 컴퓨터가 영어 중심으로 설계된 역사적·기술적 이유를 이해하고, 한국어 사용 환경에서 발생하는 문제와 올바른 대처 습관을 익힌다.

#### 📌 4-1. ASCII — 모든 것의 시작
1. ASCII(아스키) — 1963년 미국에서 만들어진 최초의 컴퓨터 문자 코드 표준
2. ASCII 오직 128자 — 영문 대소문자·숫자·특수문자만 포함, 한글 없음
3. 컴퓨터의 모든 초기 명령어·파일 시스템·프로토콜이 ASCII 기반으로 설계된 이유
4. 한글이 나중에 추가된 결과 — 인코딩 충돌(EUC-KR vs UTF-8), 한글 깨짐 현상
5. 이후 유니코드(Unicode)의 등장 — 전 세계 언어를 하나의 표준으로 통합

#### 📌 4-2. 영어가 기본인 상황들
1. 파일·폴더 이름 — 한글 이름으로 생성해도 내부적으로 처리 시 문제가 생기는 경우
2. URL(인터넷 주소) — 한글 도메인·한글 경로는 내부적으로 퍼센트 인코딩(`%ED%95%9C`)으로 변환됨
3. 터미널·명령어 — 거의 모든 명령어가 영어 약어로 구성되어 있는 이유
4. 에러 메시지 — 전 세계 공통 언어로서 영어 에러 메시지가 표준인 이유
5. 프로그래밍 언어의 키워드 — `if`, `else`, `for`, `return` 등 모두 영어 단어

#### 📌 4-3. 영어 친화 환경 만들기 실천
1. 파일명 영문화 습관 — `과제 최종본.docx` 대신 `assignment_final.docx` 사용 이유
2. 폴더 구조 영문화 — `내 문서/프로젝트` 대신 `Documents/projects` 사용 권장
3. 한글 경로 감지 방법 — 파일 탐색기 주소창에서 현재 경로 확인
4. 한글 입력 중 자동완성·검색 영향 — 검색/명령줄에서 한글-영문 전환 오작동 상황
5. 영어 타이핑 속도 향상의 중요성 — 개발자로 성장하기 위한 기초 소양으로서의 영타

- **강의 내용**: ASCII 역사, 영어 중심 설계 이유, 한글 환경 충돌 사례
- **실습 내용**: 한글 파일명 파일을 URL 에 드래그했을 때 퍼센트 인코딩으로 변환되는 것 확인, 한글 경로 감지 실습
- **과제**: 내 PC 파일·폴더 구조 중 한글로 된 이름 3개를 영문으로 변환하고 이유를 설명한 리포트 제출

### 5주차
- **대주제**: 운영체제 비교 — Windows vs macOS vs Linux
- **세부학습목표**: 세 OS의 역사와 특징을 이해하고, 같은 목표를 수행할 때 방법이 어떻게 달라지는지 비교한다.

#### 📌 5-1. 운영체제의 역사와 탄생 배경
1. Windows — DOS에서 시작하여 NT 커널로 진화한 과정, 세계 데스크톱 시장 1위
2. macOS — UNIX 기반 NeXT → OS X → macOS의 계보, Apple 하드웨어 전용 OS
3. Linux — 1991년 Linus Torvalds가 만든 오픈소스 커널, 서버 시장 사실상 독점
4. Android — Linux 커널 기반의 모바일 OS, 전 세계 스마트폰 70% 이상 사용
5. iOS — Darwin(UNIX) 기반 Apple 모바일 OS, 보안·최적화 우선 폐쇄형 생태계

#### 📌 5-2. OS별 핵심 차이 비교
1. 파일 시스템 — NTFS(Windows) vs APFS(macOS) vs ext4(Linux)의 특성 차이
2. 경로 구분자 — `\`(Windows)와 `/`(Mac·Linux)의 차이와 혼동 주의사항
3. 소프트웨어 설치 방식 — `.exe`(Win) / `.dmg`(Mac) / `apt·brew`(Linux) 비교
4. 보안 철학 — Windows는 관리자 기본 / Mac·Linux는 최소권한 원칙 기본
5. 업데이트 방식 — Windows Update / macOS Software Update / Linux 패키지 관리자

#### 📌 5-3. 내 순간에 맞는 OS 선택
1. 학교 과제·일반 업무 — Windows가 가장 넓은 소프트웨어 호환성 제공
2. 디자인·영상·음악 — macOS의 안정성과 Apple 생태계 일관성
3. 서버·개발 서버 — Linux의 안정성·가벼움·무료 라이선스
4. 개발 학습 환경 — macOS·Linux가 터미널·오픈소스와 궁합이 좋은 이유
5. WSL2(Windows Subsystem for Linux) — Windows에서 Linux 환경을 동시에 쓰는 방법

- **강의 내용**: 3대 OS의 역사, 파일 시스템·설치 방식 비교, 선택 기준
- **실습 내용**: Windows 탐색기 경로 주소창 확인 vs Mac Finder 경로 비교, WSL2 설치 후 `whoami` 명령 실행
- **과제**: 동일 작업(폴더 만들기)을 Windows GUI·cmd·WSL Linux 터미널 3가지 방법으로 수행한 비교표 제출

### 6주차
- **대주제**: 파일 시스템 완벽 이해 1 — 파일·폴더·확장자
- **세부학습목표**: 확장자가 파일 형식을 결정하는 실제 메커니즘을 체험하고, 탐색기 밖에서도 파일의 본질을 이해한다.

#### 📌 6-1. 파일의 구성 요소 이해
1. 파일 = 이름 + 확장자 + 내용 + 메타데이터(생성일·수정일·크기·권한)
2. 파일 크기와 실제 디스크 점유 크기가 다른 이유 — 클러스터(Cluster) 단위 할당
3. 숨김 파일(점으로 시작하는 파일) — `.gitignore`·`.DS_Store`·`.bashrc`의 존재 이유
4. 파일 이름의 금지 문자 목록 — OS별로 사용할 수 없는 특수문자(`\ / : * ? " < > |`)
5. 파일 메타데이터 확인 방법 — Windows `속성` / Mac `정보 가져오기`

#### 📌 6-2. 확장자의 실제 역할
1. 확장자 목록 — `.txt`·`.pdf`·`.mp4`·`.exe`·`.py`·`.json`·`.html` 각각의 역할
2. '연결 프로그램'이 결정되는 원리 — OS의 파일 연결(Association) 데이터베이스
3. 확장자를 강제로 바꿔도 내용은 바뀌지 않는 이유 — 내용과 이름의 분리
4. '알려진 파일 형식의 확장명 숨기기' 옵션이 보안 위협인 이유 — 악성코드 위장 사례
5. 동일 확장자가 여러 프로그램에서 열리는 상황 — `.csv` 파일과 엑셀·메모장 선택

#### 📌 6-3. 폴더 구조와 트리 구조
1. 폴더(디렉토리)가 계층 구조를 이루는 이유 — 트리(Tree) 자료구조
2. 루트(Root) 폴더 — `C:\`(Windows)·`/`(Mac·Linux), 모든 경로의 출발점
3. 특수 폴더의 역할 — `C:\Windows`·`C:\Users`·`/etc`·`/home`·`/usr`
4. 바탕화면·다운로드·문서 폴더가 실제로 저장되는 위치 — 단순히 바로가기가 아님
5. 폴더 구조를 손으로 트리로 스케치하는 연습 방법

- **강의 내용**: 파일 구성 요소, 확장자 역할, 디렉토리 트리 구조
- **실습 내용**: 확장자 숨기기 옵션 해제 → `.txt`를 `.mp4`로 강제 변경 → 오류 체험 → 복구
- **과제**: 교수자 제공 확장자 퀴즈(확장자로 파일 종류와 연결 프로그램 추리) 풀이 제출

### 7주차
- **대주제**: 파일 시스템 완벽 이해 2 — 경로(Path) 완전 정복
- **세부학습목표**: 절대경로와 상대경로를 암기가 아닌 논리로 이해하고, 파일 탐색기와 터미널에서 직접 이동한다.

#### 📌 7-1. 절대경로의 개념
1. 절대경로 = 루트에서 시작하는 완전한 주소 — 어디서든 동일하게 동작
2. Windows 절대경로 — `C:\Users\hojin\Documents\report.pdf`
3. macOS·Linux 절대경로 — `/home/hojin/documents/report.pdf`
4. 절대경로를 복사해서 붙여넣으면 어디서든 해당 파일을 찾을 수 있는 이유
5. 탐색기·Finder 주소창에서 현재 절대경로를 복사하는 방법

#### 📌 7-2. 상대경로의 개념
1. 상대경로 = 현재 위치를 기준으로 표기하는 경로
2. `.`(현재 디렉토리)와 `..`(부모 디렉토리)의 정확한 의미
3. `../photos/image.jpg` 형태를 그림으로 읽는 법 — 부모 폴더로 올라간 뒤 이동
4. 상대경로가 절대경로보다 유리한 상황 — 폴더를 다른 위치로 이동·공유할 때
5. 경로에 공백이 있을 때 발생하는 문제와 따옴표로 감싸는 해결법

#### 📌 7-3. 파일 탐색기로 경로 탐색 실습
1. Windows 파일 탐색기 주소창 클릭 — GUI 경로를 텍스트 경로로 전환
2. Mac Finder에서 경로 막대 표시하기 (`보기 → 경로 막대 표시`)
3. `Ctrl+L`(Windows) / `Cmd+Shift+G`(Mac) — 경로 직접 입력하여 이동
4. 최근 폴더 기록으로 빠르게 이동하는 방법
5. 경로 퀴즈 풀기 — A 위치에서 B 파일까지 상대경로 작성 연습

- **강의 내용**: 절대경로·상대경로 개념, `.`과 `..`의 의미, GUI에서 경로 탐색
- **실습 내용**: 파일 탐색기로 깊숙한 폴더까지 이동 미션, `Cmd+Shift+G` / `Ctrl+L`로 경로 직접 입력
- **과제**: 교수자 제공 경로 퀴즈 10문제(A에서 B까지 상대경로 + B의 절대경로 도출) 제출

### 9주차
- **대주제**: 인터넷과 브라우저의 동작 원리
- **세부학습목표**: 브라우저 주소창에 URL을 입력하고 결과가 화면에 나타나기까지의 과정을 단계별로 이해한다.

#### 📌 9-1. 인터넷과 웹(WWW)의 차이
1. 인터넷 = 전 세계 컴퓨터를 연결하는 물리적 네트워크 인프라
2. 웹(WWW) = 인터넷 위에서 HTTP로 동작하는 서비스 중 하나
3. 이메일·메신저·게임 서버도 인터넷 위에서 동작하지만 웹이 아닌 이유
4. HTTP(80)와 HTTPS(443) 차이 — 암호화 여부와 브라우저 자물쇠 아이콘의 의미
5. 도메인 이름(google.com)이 숫자 IP로 변환되는 DNS 개념

#### 📌 9-2. URL 구조 완전 분해
1. `프로토콜://도메인:포트/경로?쿼리#앵커` 각 구성 요소의 역할
2. 기본 포트가 숨겨지는 이유 — `https://google.com` = `https://google.com:443`
3. 한글 URL이 `%ED%95%9C%EA%B8%80`처럼 변환되는 퍼센트 인코딩의 원리
4. 쿼리스트링(`?id=1&name=kim`) — 검색·로그인 등에서 데이터를 전달하는 방식
5. 짧아진 URL(bit.ly 등)의 동작 원리 — 서버에서 실제 주소로 리다이렉트

#### 📌 9-3. 크롬 브라우저 심화 활용
1. 크롬 개발자 도구(`F12`) — Elements·Network·Console·Application 탭 개요
2. 쿠키(Cookie) — 로그인 상태를 유지하는 작은 텍스트 파일
3. 캐시(Cache) — 같은 사이트를 빠르게 로딩하기 위해 파일을 임시 저장하는 메커니즘
4. 시크릿 창 — 쿠키·캐시를 저장하지 않는 일회성 브라우저 세션
5. 브라우저 북마크·히스토리·비밀번호 관리 — 효율적인 브라우저 사용법

- **강의 내용**: 인터넷·웹·HTTP 개념, URL 구조, 쿠키·캐시·개발자 도구 개요
- **실습 내용**: 개발자 도구 Network 탭에서 사이트 접속 시 나오는 파일 목록 관찰
- **과제**: 크롬 개발자 도구에서 네트워크 요청 3개를 골라 각각의 역할 추측 설명 제출

### 10주차
- **대주제**: IP·포트·localhost — 네트워크 기초
- **세부학습목표**: IP 주소와 포트에 당황하지 않고, 인터넷 연결 원리를 생활 비유로 완벽히 납득한다.

#### 📌 10-1. IP 주소 체계 이해
1. IP 주소 = 인터넷의 집 주소 — 점으로 구분된 4개의 숫자(IPv4)
2. 공인 IP(인터넷에서 보이는 주소) vs 사설 IP(공유기 내부 주소) 차이
3. `192.168.x.x`가 항상 사설 IP인 이유 — 예약된 주소 범위
4. IPv6 등장 배경 — IPv4 주소 고갈 문제와 `::1` 표기 방식
5. `ipconfig`(Win) / `ip a`·`ifconfig`(Linux·Mac)로 내 IP 주소 확인하기

#### 📌 10-2. 포트(Port)의 역할
1. 포트 = 같은 집(IP) 안의 개별 창구 번호 (0~65535)
2. 잘 알려진 포트 — HTTP 80·HTTPS 443·SSH 22·FTP 21
3. 포트 충돌(이미 사용 중) 오류가 발생하는 원인과 해결 방법
4. 방화벽 — 허용되지 않은 포트로 들어오는 통신을 차단하는 문지기
5. 라우터(공유기)의 역할 — 여러 기기를 하나의 공인 IP로 연결하는 NAT 원리

#### 📌 10-3. localhost와 네트워크 테스트 기초
1. `localhost` = 항상 내 컴퓨터 자신을 가리키는 고정 이름(`127.0.0.1`)
2. `ping google.com` — 인터넷 연결 상태와 응답 시간(ms) 확인
3. `nslookup google.com` — 도메인 이름이 어떤 IP로 변환되는지 확인
4. 공유기 설정 화면 접속 방법 — `192.168.0.1` 또는 `192.168.1.1` 주소로 접속
5. 내 스마트폰과 같은 공유기 네트워크에서 서로의 사설 IP로 통신하는 원리

- **강의 내용**: IP 체계, 공인/사설 IP, 포트 개념, localhost, ping·nslookup 도구
- **실습 내용**: 내 공인 IP·사설 IP 각각 확인, `ping google.com` 실행 후 응답 시간 캡처
- **과제**: 내 공인 IP와 사설 IP 캡처 + 두 주소가 왜 다른지 비유를 들어 설명 제출

1. 딥페이크(Deepfake) — AI로 생성된 가짜 사진·영상을 구별하는 방법
2. AI 생성 텍스트 감지 — 자연스럽지만 정확성이 불분명한 글을 비판적으로 읽는 습관
3. 저작권과 AI — AI가 생성한 이미지·글의 저작권 귀속 문제
4. AI 검색(Perplexity·ChatGPT 검색)과 일반 검색의 차이 — 인용 출처 확인의 중요성
5. AI 도구에 의존하지 않고 기초 역량을 먼저 쌓아야 하는 이유

- **강의 내용**: LLM 기본 원리, 주요 AI 도구 비교, 올바른 활용과 디지털 리터러시
- **실습 내용**: ChatGPT에 같은 질문을 3가지 방법으로 다르게 질문 → 결과 비교 → 공식 자료로 검증
- **과제**: AI에게 물어본 내용 1개를 교차 검증하여 맞는 부분·틀린 부분을 분류하고 분석 제출

### 12주차
- **대주제**: 터미널 첫 만남 — 검은 화면과 친해지기
- **세부학습목표**: 터미널(CLI)이 무엇인지 이해하고, 기본적인 파일 조작 명령어를 손수 입력하여 검은 화면에 대한 거부감을 없앤다. (개발 도구로서의 활용은 <개발 기초> 과목에서 심화)

#### 📌 12-1. 터미널이란 무엇인가
1. GUI(그래픽) vs CLI(명령어) — 마우스로 클릭 vs 텍스트 명령어로 제어하는 방식
2. 터미널·쉘·콘솔의 차이 — 각 용어가 가리키는 것이 다름
3. Windows: cmd·PowerShell / Mac: Terminal(zsh) / Linux: bash 구분
4. 서버·원격 환경에는 GUI가 없어 CLI만으로 모든 것을 해야 하는 이유
5. CLI가 GUI보다 강력한 이유 — 마우스 5번 클릭 vs 명령어 1줄

#### 📌 12-2. 필수 기초 명령어 6개
1. `ls`(Mac·Linux) / `dir`(Windows) — 현재 폴더의 파일 목록 보기
2. `cd 경로` — 폴더 이동 / `cd ..` 부모 폴더로 / `cd ~` 홈으로
3. `pwd` — 현재 내 위치의 절대경로 확인 (Print Working Directory)
4. `mkdir 폴더명` — 새 폴더 생성 / `touch 파일명`(Mac·Linux) 빈 파일 생성
5. `clear`(Mac·Linux) / `cls`(Windows) — 화면 깨끗하게 정리

#### 📌 12-3. CLI로 파일 조작 체험
1. `cp 원본 목적지` — 파일 복사 / `mv 원본 목적지` — 파일 이동(이름 변경도 동일)
2. `rm 파일명` — 파일 삭제 주의! 휴지통이 없어 즉시 삭제됨
3. `cat 파일명` — 파일 내용 터미널에 출력
4. 경로 자동완성 — `Tab` 키로 파일·폴더 이름 자동 채우기
5. 이전 명령어 불러오기 — `↑`·`↓` 화살표 키로 명령 이력 탐색

- **강의 내용**: GUI-CLI 비교, 기초 명령어 6개, 파일 조작 명령어

#### 📌 13-1. 파일 정리 시스템 만들기
1. 폴더 구조 설계 원칙 — 날짜별·프로젝트별·종류별 중 어떤 기준이 유리한가

3. 영문 소문자·언더스코어(`_`) 또는 하이픈(`-`) 사용 — 공백 대신 구분자 사용 이유
4. 버전 관리를 파일 이름으로 — `v1`, `v2` 또는 날짜 접미사 활용법
5. 바탕화면에 파일을 쌓지 않아야 하는 이유 — OS 성능과 찾기 불편

#### 📌 13-2. 백업의 중요성과 방법
1. 3-2-1 백업 원칙 — 3개 복사본, 2가지 媒體, 1개는 오프사이트(외부) 보관
2. 클라우드 백업 — Google Drive·iCloud·OneDrive 자동 동기화 설정
3. 외장 SSD·USB 로컬 백업 — 클라우드만 믿으면 안 되는 이유(인터넷 없을 때·서비스 종료)
4. Windows 파일 히스토리 / macOS Time Machine — OS 내장 자동 백업 설정 방법
5. 백업을 확인해야 하는 이유 — "백업했다고 생각했는데 사실 안 됐을 때" 실제 사례

#### 📌 13-3. 기본 보안 습관
1. 비밀번호 관리 — 서비스마다 다른 비밀번호, 비밀번호 관리자(Bitwarden 등) 활용
2. 피싱(Phishing) 이메일 감지 — 발신자 주소 확인, 첨부 파일 의심, URL 미리 보기
3. 공공 와이파이 사용 시 주의 — 중간자 공격(MITM) 위험과 VPN의 역할
4. 윈도우 업데이트를 미루면 안 되는 이유 — 보안 취약점 패치의 중요성
5. 의심스러운 프로그램 설치 전 확인 사항 — 공식 사이트 여부·서명 확인·바이러스 토탈 활용

- **강의 내용**: 파일 정리 규칙, 3-2-1 백업 원칙, 기본 보안 습관
- **실습 내용**: 본인 PC 바탕화면·다운로드 폴더 정리 → 폴더 구조 재설계 → 클라우드 백업 자동 동기화 설정
- **과제**: 내 PC의 파일 정리 현황 '비포·애프터' 캡처 + 앞으로 사용할 폴더 구조 설계안 제출
